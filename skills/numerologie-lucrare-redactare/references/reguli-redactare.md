# Reguli de redactare

## Surse si prioritate

1. Datele confirmate de utilizator.
2. Iesirea JSON din `scripts/calculator_numerologic_examen.py` pentru toate valorile numerologice.
3. Template-ul selectat din `assets/`.
4. Lucrarile Roman Andreea Maria si Birsan Daniel Robert ca repere de stil si nivel de detaliere.
5. Skill-urile SVG dedicate pentru diagrame, alimentate cu valorile calculatorului.

Nu copia date, rezultate sau formulari personale dintr-o lucrare in alta.
Nu consulta documentatia numerologica pentru a recalcula sau suprascrie rezultatele calculatorului inclus.

## Stil

- Conversational: adreseaza persoana direct, explica accesibil si foloseste analogii concrete.
- Formal: pastreaza claritatea si amploarea, dar foloseste persoana a treia si un ton sobru.
- Intra direct in definitie sau interpretare. Nu folosi formulari metatextuale precum `[Prenume], aici ne uitam la...`, `aici vedem...`, `in aceasta sectiune analizam...` ori `urmeaza sa vedem...`.
- Foloseste adresarea directa numai cand transmite o interpretare personala, nu pentru a anunta tema sau a descrie actul analizei.
- Foloseste arhetipuri numai pentru Vibratia interioara, Vibratia exterioara si Vibratia globala. Nu numi si nu sugera roluri arhetipale in interpretarile celorlalte vibratii, ale destinului, numelui, matricilor, ciclurilor, relatiilor, spiritului sau ajutoarelor.
- La vibratiile fundamentale, traseul de reducere se interpreteaza conversational numai cand exista o reducere suplimentara. Se explica direct persoanei cifrele initiale, suma intermediara, cifrele reducerii si rezultatul final. Daca valoarea este deja o singura cifra, se interpreteaza direct, fara a mentiona absenta traseului. Pentru 0 se foloseste formularea prudenta ca nu adauga o directie separata, dar poate deschide sau amplifica potentialul cifrei alaturate.
- Interpretarea traseului nu se repeta apoi sub alte formulari. Accentul ramane pe rezultatul final, iar paragrafele care urmeaza trebuie sa aduca pe rand manifestare observabila, exemplu practic, rutina zilnica sau saptamanala, umbra si criteriu de maturizare. Amploarea se pastreaza prin informatie noua, nu prin reiterare.
- La Vibratia globala se interpreteaza intotdeauna combinatia dintre interior si exterior. Traseul de reducere se explica separat numai cand suma lor este mai mare decat 9.
- Nu extinde aceasta obligatie la Calea Destinului, Destin sau Aspectele de indreptat. Puntile pastreaza lectura valorilor initiale si a rezultatului, iar la Solutia aspectelor de indreptat se recomanda interpretarea rezultatului.
- Scara bunastarii este subcapitol separat al Structurii matriciale, dupa comparatia cu optimul si analogia cai-trasura-vizitiu. Primeste titlul numerotat `4.6. Scara bunastarii`, ancora proprie si index `SUB` in variantele de revizie.
- Numarul neamului include obligatoriu calculul in intervalul 1-22 si un tabel cu arcana rezultata si interpretarea ampla a resursei mostenite, manifestarii concrete, umbrei si maturizarii.
- Spirit -> Inclinatii profesionale include obligatoriu calculele DA si NU pe randuri distincte si un tabel cu arcana fiecarui rezultat; DA descrie aplicabilitatea profesionala, iar NU obstacolul si cheia de lucru.
- Evita frazele categorice, promisiunile predictive si exagerarile ezoterice.
- Creeaza imagini din cuvinte numai cand clarifica sensul: casa, pod, atelier, drum, arhitect, dirijor.
- Leaga fiecare interpretare de datele persoanei si de celelalte rezultate relevante.

## Consecventa documentului

- Include `Cuvant inainte` dupa Cuprins si inainte de Capitolul 1, folosind textul reutilizabil din `vault/Numerologie/Introducere.md`. In lucrare, acesta este singurul titlu afisat pentru sectiune.
- Pastreaza metafora calatoriei si imaginea din cuvinte care leaga conceptele intr-o harta usor de urmarit. Modifica selectiv numai formularile care trebuie corectate sau aliniate metodologic; nu rescrie textul fara nevoie.
- Integreaza in aceeasi poveste ciclicitatile, anii importanti interiori si exteriori, ciclurile de 7, 9 si 12 ani, Soarta si Destinul, lectiile de viata si anul personal. Nu le transforma intr-o enumerare tehnica.
- Datele generale din Markdown si HTML trebuie sa fie identice.
- Daca exista relatie, include numele complet, prenumele activ, data, genul si tipul relatiei, in masura in care au fost furnizate.
- In capitolul `Relatii`, imediat dupa titlul capitolului si inainte de primul subcapitol, prezinta un bloc strict factual cu numai trei campuri: numele complet al celeilalte persoane, data nasterii si tipul relatiei. Interpretarea incepe abia in subcapitole.
- Daca in datele persoanei nu este declarata nicio relatie, elimina integral capitolul `Relatii` si intrarea lui din cuprins; nu genera campuri goale sau texte generice.
- Foloseste denumiri complete pentru concepte; nu folosi prescurtari de vibratii.
- HTML-ul final pastreaza clasele de prezentare, stilurile, tabelele si structura template-ului, dar nu pastreaza indexurile de revizie.
- In varianta finala, elimina indexurile vizibile numai cand utilizatorul cere trecerea din revizie in final.
- La trecerea din revizie in final, numarul versiunii ramane identic si se schimba numai sufixul: `v1.07r` devine `v1.07f`. Finala se deriva din revizia validata, nu dintr-o finala mai veche.

## Intervale de ani

- `complet (0-108 ani)`: calculeaza si prezinta intregul interval cerut de metoda.
- `specific`: calculeaza numai intervalul confirmat, dar pastreaza contextul ciclului necesar interpretarii.
- Nu presupune automat anul curent drept inceputul intervalului.
