# Reguli de redactare

## Surse si prioritate

1. Datele confirmate de utilizator.
2. Iesirea JSON din `scripts/calculator_numerologic_examen.py` pentru toate valorile numerologice.
3. Template-ul selectat din `assets/`.
4. Lucrarile Roman Andreea Maria si Birsan Daniel Robert ca repere de stil si nivel de detaliere.
5. Skill-urile SVG dedicate pentru diagrame, alimentate cu valorile calculatorului.

Nu copia date, rezultate sau formulari personale dintr-o lucrare in alta.
Nu consulta documentatia numerologica pentru a recalcula sau suprascrie rezultatele calculatorului inclus.

## Stil

- Conversational: adreseaza persoana direct, explica accesibil si foloseste analogii concrete.
- Formal: pastreaza claritatea si amploarea, dar foloseste persoana a treia si un ton sobru.
- Evita frazele categorice, promisiunile predictive si exagerarile ezoterice.
- Creeaza imagini din cuvinte numai cand clarifica sensul: casa, pod, atelier, drum, arhitect, dirijor.
- Leaga fiecare interpretare de datele persoanei si de celelalte rezultate relevante.

## Consecventa documentului

- Datele generale din Markdown si HTML trebuie sa fie identice.
- Daca exista relatie, include numele complet, prenumele activ, data, genul si tipul relatiei, in masura in care au fost furnizate.
- Foloseste denumiri complete pentru concepte; nu folosi prescurtari de vibratii.
- HTML-ul final pastreaza clasele, indexurile de revizie, stilurile, tabelele si structura template-ului.
- In varianta finala, elimina indexurile vizibile numai cand utilizatorul cere trecerea din revizie in final.

## Intervale de ani

- `complet (0-108 ani)`: calculeaza si prezinta intregul interval cerut de metoda.
- `specific`: calculeaza numai intervalul confirmat, dar pastreaza contextul ciclului necesar interpretarii.
- Nu presupune automat anul curent drept inceputul intervalului.
